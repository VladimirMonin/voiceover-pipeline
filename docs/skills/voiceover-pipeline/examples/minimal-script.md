# Пример минимального Markdown-сценария

> АГЕНТ: ЧИТАЙ ЭТОТ ФАЙЛ ЦЕЛИКОМ.
> Используй как образец при создании `script.md` для пользователя.

## Структура

Сценарий разбивается на чанки строкой `******` (шесть звёздочек).
Каждый непустой блок становится отдельным MP3-файлом.

```markdown
Первый фрагмент сценария. Это вступление к видео или подкасту.
Здесь можно описать, о чём пойдёт речь.

******

Второй фрагмент. Основная часть — здесь раскрывается тема.
Можно использовать несколько предложений. TTS прочитает их
естественным образом, делая паузы между предложениями.

******

Третий фрагмент. Ещё одна часть основного содержания.
Каждый чанк озвучивается отдельно, поэтому можно разбивать
длинный текст на логические блоки.

******

Четвёртый фрагмент — заключение. Подведение итогов,
выводы и завершающие слова.
```

## Правила

- Разделитель по умолчанию: `******`
- Не оставляйте пустых блоков — они игнорируются
- Для Remotion: один чанк ≈ одна сцена
- Рекомендуемый размер чанка: до 2000 символов
- Избегайте Markdown-разметки в тексте для озвучки (TTS её игнорирует)
- Числа могут читаться не так как ожидается — валидатор предупредит

## Валидация

```powershell
voiceover validate --script "script.md" --json
```
